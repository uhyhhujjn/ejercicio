  • Página Web Personal o Portafolio: Página Web Personal o Portafolio Descripción: Crea un sitio web personal o un portafolio en el que muestres tus habilidades, experiencia y proyectos. Puedes utilizar HTML, CSS, y JavaScript, y alojarlo en GitHub Pages. Objetivo: Practicar el desarrollo web y la gestión de un proyecto desde su creación hasta la publicación en GitHub Pages.
    Tecnologías: Blog con Jekyll y GitHub Pages.
    Descripción: Configura un blog personal utilizando Jekyll, un generador de sitios estáticos. Jekyll es compatible con GitHub Pages, lo que facilita la creación y gestión de contenido.

    Objetivo: Aprender a trabajar con un generador de sitios estáticos y manejar la publicación automática en GitHub.
    Tecnologías: Jekyll, Markdown, GitHub Pages.


    • Consejos para el Uso de Git y GitHub Commits Frecuentes:


    Realiza commits frecuentes con mensajes claros y descriptivos. Branching: Utiliza ramas (branches) para trabajar en nuevas funcionalidades o corregir errores sin afectar el código principal. Pull Requests: Si trabajas en equipo o contribuyes a un proyecto de código abierto, realiza pull requests para que otros revisen y fusionen tus cambios. Documentación: Asegúrate de que cada proyecto tenga un archivo README.md que explique cómo instalar y utilizar el proyecto. Estos proyectos no solo te permitirán mejorar tus habilidades técnicas, sino que también te ayudarán a desarrollar un portafolio robusto que puedes mostrar a potenciales empleadores o colaboradores.
