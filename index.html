¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite a múltiples desarrolladores trabajar en un proyecto de manera colaborativa. Permite realizar un seguimiento de los cambios en el código fuente, revertir versiones y gestionar ramas para el desarrollo paralelo.

¿Qué es GitHub?

GitHub es una plataforma de alojamiento de repositorios Git que facilita la colaboración en proyectos de software. Proporciona una interfaz web que permite gestionar repositorios, crear ramas, realizar revisiones de código y abrir pull requests para fusionar cambios.

¿Por qué usar Git y GitHub?

  • Colaboración: Facilita el trabajo en equipo al permitir que múltiples personas trabajen en un proyecto simultáneamente.
  • Historial de cambios: Mantiene un registro completo de todos los cambios realizados en el proyecto.
  • Ramas: Permite crear y trabajar en diferentes ramas de desarrollo sin afectar el código principal.
  • Revisión de código: Facilita la revisión y discusión de cambios a través de pull requests.

  • git init - Inicializa un nuevo repositorio Git.
  • git clone [url] - Clona un repositorio existente de GitHub en tu máquina local.
  • git add [archivo] - Añade un archivo al área de preparación (stage) para ser confirmado.
  • git commit -m "mensaje" - Confirma los cambios con un mensaje descriptivo.
  • git push - Envía los cambios confirmados a un repositorio remoto en GitHub.
  • git pull - Obtiene y fusiona cambios desde el repositorio remoto a tu copia local.
